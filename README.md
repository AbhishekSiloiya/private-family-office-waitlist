## Waitlist Form Setup

GitHub Pages is static and cannot store emails by itself.

The waitlist form currently posts to Formspree:

```html
<form class="form-panel" action="https://formspree.io/f/mzdodkog" method="POST">
```

The form sends:

- `email` — subscriber email address
- `subject` — notification subject for Formspree
- `source` — hidden source label for tracking
- `_gotcha` — hidden honeypot field for basic bot filtering

Recommended Formspree dashboard settings before launch:

- verify the target email address
- enable email notifications
- enable spam protection or reCAPTCHA
- restrict submissions to the published GitHub Pages domain once live
- optionally connect the Google Sheets plugin for a spreadsheet export
